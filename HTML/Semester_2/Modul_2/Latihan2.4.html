Pendahuluan

Pada zaman sebelum terciptanya OSI untuk melakukan sebuah komunikasi pada jaringan komputer tidak mudah seperti sekarang ini.
Di masa itu menggunakan protokol jaringan masing-masing, sehingga untuk melakukan pertukaran data susah untuk dilakukan.
Open System Interconnection atau lebih dikenal dengan sebutan OSI merupakan perkembangan teknologi di dalam jaringan komputer.
Model OSI menyediakan secara konseptual kerangka kerja untuk komunikasi antar komputer.
Di dalam konteks jaringan data, sebuah protokol adalah aturan formal dan kesepakatan yang menentukan bagaimana komputer bertukar informasi melewati sebuah media jaringan.
Sebuah protokol mengimplementasikan salah satu atau lebih dari lapisan â€“ lapisan OSI.
Model lapisan layer terbagi menjadi dua layer yaitu, layer atas dan bawah.
Layer atas biasanya diimplementasikan software dan layer bawah diimplementasikan sebagai software atau hardware.

Pembahasan

Model OSI dibagi menjadi tujuh layer, tiap layer tersebut memiliki peran masing-masing dan saling berkaitan antar layer diatas dengan layer yang berada dibawahnya. OSI secara konseptual terbagi menjadi 7 lapisan yaitu, physical layer, data link layer, network layer, transport layer, session layer, presentation layer, dan application layer.
Ketujuh layer tersebut mempunyai prinsip-prinsip yang digunakan, sebuah layer harus dibuat bila diperlukan tingkat abstraksi yang berbeda, setiap layer harus memiliki fungsi tertentu dan berbeda, fungsi setiap layer harus dipilih dengan teliti sesuai ketentuan standar protokol internasional, pada batas-batas layer diusahakan meminimalkan aliran informasi yang melewati interface, dan jumlah layer harus cukup banyak sehingga fungsi-fungsi yang berbeda tidak perlu disatukan dalam satu layer di luar keperluannya.
Akan tetapi jumlah layer juga harus diusahakan tidak terlalu banyak (sedikit mungkin) agar arsitektur jaringan tidak menjadi kompleks atau sulit dipakai.

Kesimpulan

Dalam cara kerja nya data dikirim dari titik sumber ke titik tujuan. Data akan dialirkan dari application layer sampai dengan physical layer.
Setelah sampai physical layer data akan berubah menjadi bits, dan kemudian akan ditransmisikan dengan menggunakan media jaringan.
Di Sisi penerima, data akan diterima dalam bentuk bits di physical layer. Selanjutnya bits tersebut akan diubah menjadi data link layer.
Data link layer akan memeriksa kesalahan yang mungkin terjadi pada saat proses transmisi data dan membungkus bit kedalam bentuk data frame.
